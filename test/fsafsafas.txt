X: Load Balancers
Application load balancers - Layer 7 (HTTP/HTTPS)It looks inside the packet. It can route `/api` to Server A and `/images` to Server B. It handles SSL termination and is best for websites/APIs. 
Network load balancers: Layer 4 (TCP/UDP) - Fast. It doesn't look inside. It just forwards packets at lightning speed. It handles millions of requests per second. Used for gaming or extreme performance. 
Gateway load balancers: Layer 3
Algorithms: Round Robin , Least Connections, IP Hash: User A always goes to Server 1 (Sticky). 

X: OSI
1. Physical: Sending raw bits (1s and 0s) over a physical medium.
2. Data: Moves "Frames" between devices on the SAME network
3. Network:  Moves "Packets" between different networks.
4. Transport: Reliability and error checking. Chops data into "Segments". (TCP/UDP)
5. Session: Starts, keeps, and ends the conversation between devices.
6. Presentation: Formatting, encryption, and compression. It translates data into a readable format.
7. Application: This is what you see. It creates the data and talks to the user.

1. How does DNS work?

2. What is a Security Group?
A virtual firewall in AWS that controls inbound and outbound traffic for EC2 instances at the instance level.

3. What is VLAN?
Virtual LAN logically separates network segments within the same physical switch, improving security and reducing broadcast domains.

4. What is a Subnet and why do we need it?
A subnet divides a VPC/network into smaller segments. It helps organize and isolate resources for performance, security, and routing control.

13. What is the difference between application load balancer and network load balancer give me example
An Application Load Balancer (ALB) operates at Layer 7 (HTTP/HTTPS) to route traffic based on content (URLs, headers), ideal for web apps
A Network Load Balancer (NLB) operates at Layer 4 (TCP/UDP), forwarding traffic based on IP/port, ideal for high-performance, low-latency needs
